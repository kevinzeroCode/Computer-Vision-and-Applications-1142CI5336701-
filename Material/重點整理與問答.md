# Lecture 01 — Introduction

## 重點整理

### 1. 課程框架
- CV 問題分三個層次：**Single view**（辨識/追蹤）、**Two views**（立體視覺/深度）、**Multi views**（3D 重建）
- 標準解題流程：定義問題 → 假設條件 → 評估可行性 → 選工具 → 驗證調整

### 2. 四大平面變換（由少到多自由度）

| 類型 | DOF | 矩陣形式 | 保留不變量 |
|------|-----|----------|-----------|
| **Euclidean** | 3 | `[R t; 0 1]` | 距離、角度、面積 |
| **Similarity** | 4 | `[sR t; 0 1]` | 長度比、角度、面積比 |
| **Affine** | 6 | `[A t; 0 1]` | 平行線、面積比、長度比（平行方向） |
| **Projective** | 8 | `[h11..h33]` | Cross-ratio（四點交比） |

### 3. 相機投影簡介
- 透視投影公式：`x = fX/Z`，`y = fY/Z`
- 3D 點 → 外參 → 內參 → 2D 像素

### 4. 電腦視覺產業現況
- 市場規模（2016: \$1.1B → 2025: \$26.2B）
- 主流技術演進：2015 CNN → 2021 ViT → 2024 Diffusion/NeRF/3D Gaussian Splatting
- 應用領域：自駕車、醫療影像、製造業檢測、零售、農業、安全監控

## 問答 10 題

**Q1. 電腦視覺的「Single view」、「Two views」、「Multi views」各自能解決什麼問題？**
> - Single view：物體辨識、偵測、追蹤、AR 疊加、影像拼接
> - Two views：立體深度感知、雙眼視差、機器人導航
> - Multi views：3D 結構重建、Image-based rendering、SfM

**Q2. Projective transformation 有幾個 DOF？為何需要至少 4 組對應點？**
> 8 DOF。每組對應點提供 2 個方程式，4 組共 8 個方程式剛好可解出 8 個未知數（H 矩陣共 9 元素，差一個 scale 固定）。

**Q3. Affine 與 Projective 變換最大的差異是什麼？**
> Affine 保留平行線（矩陣最後一列是 `[0 0 1]`），Projective 不保留平行線，平行線在 Projective 下會相交於消失點（vanishing point）。

**Q4. Similarity transformation 保留哪些不變量？**
> 長度比（ratios of length）、角度（angles）、面積比（ratios of areas）。不保留絕對距離（因有 scale s）。

**Q5. 課程中強調學生要成為哪種角色？為什麼？**
> Developer（開發者）或 Problem solver（探索者），而非只是 User（使用 library 的人）。因為只會用工具的人可替代性高，理解演算法背後原理才有核心價值。

**Q6. Cross-ratio（交比）是什麼類型變換的不變量？為何重要？**
> Projective transformation 的不變量。重要性在於它是 2D projective geometry 中唯一的純投影不變量，可用來比對從不同視角拍攝的同一場景。

**Q7. 作業要求使用 AI 工具可以嗎？需要注意什麼？**
> 可以，老師明確說 "Use of AI tools is acceptable"，但必須 **disclose（揭露）**使用情況，否則涉及學術誠信問題。

**Q8. 什麼是 Structure from Motion（SfM）？屬於哪個層次的 CV 問題？**
> 從多張 2D 圖片推算相機運動軌跡與場景 3D 結構的方法。屬於 Multi views 層次。

**Q9. 電腦視覺最新趨勢（2024~2026）中，哪些技術正在崛起？**
> - **3D Gaussian Splatting**（即時 3D 渲染）
> - **Diffusion Models**（生成式影像）
> - **Vision-Language Models (VLMs)**（多模態）
> - **Edge AI**（端側即時推論）
> - **NeRF**（神經輻射場 3D 重建）

**Q10. Euclidean transformation 的矩陣結構為何？它保留什麼？**
> 矩陣形式：`[R t; 0 1]`，其中 R 為旋轉矩陣（R^T R = I），t 為平移。
> 保留：絕對距離、角度、面積（形狀和大小完全不變）。

# Lecture 02 — Pinhole Camera

## 重點整理

### 1. 影像感測器
- **CCD/CMOS**：彩色感測器需搭配 Bayer Pattern（每 4 像素：2G + 1R + 1B）
- **Color bit**：8bit=256階；10bit=1024階；12bit=4096階；14bit=16384階
- **RAW data**：感測器直接輸出，未經處理
- **Rolling shutter**：逐行曝光，移動物體會產生扭曲；Global shutter 全部同時曝光

### 2. 針孔相機模型（Pinhole Camera）
- 透視投影：`x' = f·X/Z`，`y' = f·Y/Z`
- 正射投影（Orthographic）：`x' = x`，`y' = y`（工程製圖用）
- 兩大特性：遠小近大、平行線會匯聚於消失點（vanishing point）

### 3. 內參矩陣 K（Intrinsic Parameter）

**理想相機（簡化）：**
```
K = | f  0  px |
    | 0  f  py |
    | 0  0  1  |
```

**實際相機（完整）：**
```
K = | fx  γ  px |
    | 0  fy  py |
    | 0   0   1 |
```
- `fx`, `fy`：x/y 方向焦距（像素單位）
- `(px, py)`：主點（Principal point），影像中心偏移
- `γ`：Skew（感測器軸不垂直，現代相機通常 ≈ 0）

### 4. 外參矩陣（Extrinsic Parameter）
- 定義相機在世界座標系中的位置與方向
- `X_cam = [R | t] · X_world`
  - R：3×3 旋轉矩陣（R^T R = I）
  - t：3×1 平移向量

### 5. 完整相機模型
```
x_img = K · [R | t] · X_world
      = K · R · T · X_world
```
**兩階段：** 外參（世界→相機3D） → 內參（相機3D→影像2D）

### 6. 鏡頭畸變（Lens Distortion）
- **Barrel distortion**（桶形）：邊緣向內縮
- **Pincushion distortion**（枕形）：邊緣向外凸
- 校正公式（徑向畸變）：
  `x_d = (1 + k0·r² + k1·r⁴ + k4·r⁶)·x_s + ...`

---

## 問答 10 題

**Q1. 相機模型的完整公式是什麼？各符號代表什麼？**
> `x_img = K · R · T · X_world`
> - K：內參矩陣（焦距、主點、Skew）
> - R：旋轉矩陣（外參，相機朝向）
> - T：平移矩陣（外參，相機位置）
> - X_world：世界座標中的 3D 點

**Q2. 內參矩陣 K 有哪些參數？各自代表什麼物理意義？**
> - `fx`, `fy`：x/y 軸焦距（影響影像放大倍率）
> - `(px, py)`：主點，即光軸與影像平面的交點（通常接近影像中心）
> - `γ`：Skew，感測器兩軸不垂直時才不為 0，現代相機幾乎為 0

**Q3. 外參與內參的本質差異是什麼？**
> - 外參（Extrinsic）：描述相機在 **3D 空間中的位置與方向**（世界座標 → 相機座標，3D→3D）
> - 內參（Intrinsic）：描述 **3D 如何投影到 2D 影像**（相機座標 → 像素座標，3D→2D）

**Q4. 為什麼真實相機需要用多項式修正鏡頭畸變？**
> 真實鏡頭不是理想薄透鏡，邊緣光線彎曲程度與中心不同，導致 Barrel（桶形）或 Pincushion（枕形）畸變。畸變量與距主點距離 r 有關，需用 `k0·r² + k1·r⁴ + ...` 的多項式描述。

**Q5. Rolling Shutter 是什麼？會造成什麼問題？如何減輕？**
> 感測器**逐行**讀取像素，不同行的曝光時間點不同。移動物體或快速搖晃相機時會產生影像扭曲（斜切、果凍效果）。
> 減輕方法：提高快門速度，或使用 Global Shutter 感測器。

**Q6. 透視投影（Perspective）與正射投影（Orthographic）的差異？各用在哪裡？**
> - 透視投影：`x' = fX/Z`，近大遠小，有透視效果，用於**一般相機、電腦視覺**
> - 正射投影：`x' = x`，遠近大小相同，用於**工程製圖、CAD 軟體**

**Q7. Bayer Pattern 是什麼？為什麼 G 像素最多？**
> 彩色 CCD/CMOS 感測器上的濾色排列，每 4 個像素包含 **2G + 1R + 1B**。
> G 最多是因為人眼對綠色最敏感，多取樣綠色可提升感知亮度解析度。

**Q8. 主點（Principal point）不在影像中心會有什麼影響？**
> 影像中心與光軸的交點偏移，導致影像幾何計算出現系統性偏差。在 K 矩陣中需加入 `(px, py)` 補償，否則重投影誤差會很大。

**Q9. 電腦圖學（CG）與電腦視覺（CV）對外參的定義有什麼差異？**
> - CG：4×4 矩陣，描述「物體放到哪裡」（世界→相機是正向操作）
> - CV：3×4 矩陣（第 4 列為虛設），**是 CG 的逆矩陣**，描述「相機看世界的角度」

**Q10. 什麼是焦距（focal length）？焦距變大/變小分別代表什麼？**
> 焦距 f 是鏡頭光學中心到影像平面的距離。
> - f 增大（望遠鏡頭）：視角（FOV）變窄，遠處物體放大
> - f 減小（廣角鏡頭）：視角變寬，影像包含更多場景，邊緣畸變可能增大

# Lecture 03 — Projective 2D Geometry

## 重點整理

### 1. 齊次座標（Homogeneous Coordinates）
- 2D 點 `(x, y)` → 齊次表示 `(x, y, 1)ᵀ` 或 `(kx, ky, k)ᵀ`（k≠0）
- 直線 `ax + by + c = 0` → 向量 `l = (a, b, c)ᵀ`
- **點在直線上的條件：`xᵀl = 0`**

### 2. 點與直線的基本運算
| 運算 | 公式 |
|------|------|
| 兩直線交點 | `x = l × l'` |
| 兩點連線 | `l = x × x'` |
| 無窮遠點（平行線交點） | `x = (b, -a, 0)ᵀ` |
| 無窮遠線 | `l∞ = (0, 0, 1)ᵀ` |

### 3. 對偶性（Duality）
- 2D 射影幾何中，**點** 與 **線** 完全對偶
- 任何關於點線的定理，交換點↔線可得對偶定理
- 例：「兩點決定一線」⟺「兩線決定一點」

### 4. Conics（二次曲線）
- 方程式：`ax² + bxy + cy² + dx + ey + f = 0`
- 矩陣形式：`xᵀCx = 0`（C 為 3×3 對稱矩陣）
- **5 個點決定一條 conic**
- 切線：過 conic 上點 x 的切線為 `l = Cx`
- **Dual conic（對偶二次曲線）**：`lᵀC*l = 0`，其中 `C* = C⁻¹`

### 5. Homography（單應矩陣）
- 定義：`x' = Hx`（H 為 3×3 非奇異矩陣）
- **8 DOF**，需至少 **4 組對應點**（每組貢獻 2 個方程）
- 僅適用於**平面場景**（或純旋轉相機）
- 不需要知道 K、R、t！

**變換後各幾何對象的公式：**
```
點：  x' = Hx
直線：l' = H⁻ᵀl
Conic：C' = H⁻ᵀ C H⁻¹
```

### 6. 四大變換類型（由特殊到一般）

| 類型 | DOF | 矩陣結構 | 不變量 |
|------|-----|----------|--------|
| **Isometry（等距）** | 3 | `[R t; 0 1]` | 長度、角度、面積 |
| **Similarity（相似）** | 4 | `[sR t; 0 1]` | 長度比、角度、面積比 |
| **Affine（仿射）** | 6 | `[A t; 0 1]` | 平行線、面積比、平行長度比 |
| **Projective（射影）** | 8 | `[A t; v 1]` | Cross-ratio（交比） |

### 7. 無窮遠點與消失點
- Affine 變換：無窮遠線 `l∞` 保持在無窮遠
- Projective 變換：無窮遠線 **可以變成有限距離**（這就是透視消失點的由來）

### 8. 射影變換的分解
```
H = H_S · H_A · H_P
  = [sR t; 0ᵀ 1] · [K 0; 0ᵀ 1] · [I 0; vᵀ v]
```
相似 × 仿射 × 純射影

---

## 問答 10 題

**Q1. Homography 矩陣有幾個 DOF？為什麼需要至少 4 組對應點？**
> 8 DOF（3×3 矩陣共 9 元素，差 1 個 scale）。
> 每組對應點提供 2 個線性方程，4 組 = 8 方程剛好解出 8 個未知數。

**Q2. 什麼情況下可以用 Homography？什麼情況下不能用？**
> **可以用**：3D 場景是平面、或相機只有旋轉（無平移）
> **不能用**：場景有深度差異（非平面），此時需用 Fundamental/Essential Matrix 處理

**Q3. 齊次座標中，兩條平行線的交點在哪裡？說明其幾何意義。**
> 兩平行線 `l=(a,b,c)` 和 `l'=(a,b,c')` 的交點為 `(b, -a, 0)ᵀ`，第三分量為 0，代表**無窮遠點（point at infinity）**。幾何意義：平行線在透視投影下會匯聚於消失點，即攝影中的 vanishing point。

**Q4. 什麼是 Duality（對偶性）？舉一個對偶定理的例子。**
> 2D 射影幾何中，「點」與「線」互換後所有定理仍成立。
> 例子：「兩點決定唯一直線（l = x × x'）」↔ 「兩線決定唯一交點（x = l × l'）」

**Q5. Conic（二次曲線）需要幾個點來確定？為什麼？**
> 需要 **5 個點**。Conic 有 6 個係數 `(a,b,c,d,e,f)`，差一個 scale，有 5 個自由度。每個點提供 1 個約束方程，5 個點剛好確定唯一 conic。

**Q6. Affine 變換與 Projective 變換對無窮遠線的影響有何不同？**
> - **Affine**：無窮遠線 `l∞=(0,0,1)ᵀ` 在變換後仍在無窮遠（矩陣最後一列是 `[0 0 1]`），平行線保持平行
> - **Projective**：無窮遠線可以映射到有限位置，平行線會相交 → 產生透視消失點效果

**Q7. 直線經 Homography 變換後的公式是什麼？為什麼不是直接乘 H？**
> `l' = H⁻ᵀ l`（H 的逆轉置）
> 因為直線是 **covariant**（協變），點是 **contravariant**（逆變）。保持 `x'ᵀl' = 0` 需要：
> `(Hx)ᵀ(H⁻ᵀl) = xᵀH ᵀH⁻ᵀl = xᵀl = 0` ✓

**Q8. Isometry 與 Similarity 的差異是什麼？多了什麼自由度？**
> Similarity = Isometry + **Scale（縮放）**，多了 1 個 DOF（共 4 DOF）。
> Isometry 保留絕對距離；Similarity 只保留距離**比例**，不保留絕對大小。

**Q9. Cross-ratio（交比）是什麼？為何是 Projective 的不變量？**
> 四個共線點 A, B, C, D 的交比：`(AC/BC) / (AD/BD)`
> 即「比值的比值」（ratio of ratios）。
> 它是唯一在 Projective 變換下保持不變的量，可用於不同視角影像間的幾何比對。

**Q10. Projective 變換可以分解為哪三個部分？各部分代表什麼？**
> `H = H_Similarity × H_Affine × H_Projective`
> - `H_S`（Similarity）：旋轉 + 等向縮放 + 平移（保形狀）
> - `H_A`（Affine）：非等向縮放 + 剪切（讓平行線保持平行的部分）
> - `H_P`（純 Projective）：透視扭曲部分（讓無窮遠線映射到有限位置）
